Overview

프로젝트 소개

Mini Game Hub는 여러 개의 짧은 미니게임을 한 사이트 안에서 선택해 플레이할 수 있도록 만든 프로젝트입니다. 사용자가 앱을 설치하지 않아도 웹 브라우저에서 바로 게임을 실행할 수 있게 만드는 것이 목표입니다.

Purpose

제작 목적

단순한 웹페이지 제작을 넘어서 JavaScript로 실제 게임 로직을 구현해보기 위해 시작했습니다. 화면 전환, 점수 계산, 타이머, 충돌 판정, 승패 처리, 모바일 조작 같은 기능을 직접 구현하면서 게임 개발과 웹 개발을 함께 연습하는 것이 목적입니다.

Tech Stack

사용 기술

HTML

게임 선택 화면, 게임 페이지, 버튼, 점수판 등 전체 구조를 구성했습니다.

CSS

모바일 중심 레이아웃, 게임 카드, 조이스틱, 버튼, 게임 보드 디자인을 구현했습니다.

JavaScript

게임 시작, 타이머, 점수 계산, 승패 판정, 키보드/터치 입력, 멀티플레이 로직을 구현했습니다.

Mobile Web

스마트폰에서 플레이하기 좋도록 터치 조작과 화면 비율을 중심으로 설계했습니다.

Features

주요 기능

  • 싱글플레이 미니게임 목록
  • 한 기기에서 두 명이 함께 플레이하는 로컬 멀티플레이 게임
  • 모바일 중심 게임 화면
  • 게임별 시작 화면과 결과 화면
  • 점수 계산 및 승패 판정
  • 터치 조작 최적화
  • 게임 카드 기반 선택 UI
  • 데이터베이스 없이 작동하는 정적 웹 구조
Game List

구현한 게임

Single Player

반응속도 테스트, 클릭 게임, 숫자야구, 카드 뒤집기 등 혼자 플레이하는 게임들을 추가했습니다.

Multiplayer

2인용 퐁, 턴제 카드 뒤집기, 계산 대결, 영역 점령전 등 한 기기에서 두 명이 함께 플레이하는 게임을 구현하고 있습니다.

Current Progress

현재는 싱글플레이와 멀티플레이 게임을 계속 추가하면서, 최종적으로 각각 10개씩 구성하는 것을 목표로 하고 있습니다.

Development Process

개발 과정

초기 구조 제작

메인 허브 화면과 게임 카드 구조를 만들고, 각 게임 페이지로 이동할 수 있는 기본 구조를 제작했습니다.

싱글플레이 게임 추가

반응속도, 클릭, 숫자야구, 카드 뒤집기처럼 혼자서 플레이할 수 있는 게임들을 추가했습니다.

멀티플레이 게임 추가

2인용 퐁과 턴제 게임처럼 한 기기에서 두 명이 같이 플레이할 수 있는 게임을 구현했습니다.

모바일 조작 개선

스마트폰에서 스크롤, 확대, 터치 이동이 게임 조작을 방해하지 않도록 모바일 터치 처리를 개선했습니다.

Problem Solving

어려웠던 점과 해결 방법

문제점 1

모바일에서 게임을 플레이할 때 브라우저의 스크롤, 확대, 터치 제스처가 게임 조작을 방해하는 문제가 있었습니다.

해결 방법

게임 페이지에서는 터치 동작을 제한하고, 불필요한 스크롤과 확대를 막는 방식으로 조작감을 개선했습니다.

문제점 2

멀티플레이 게임은 한 화면에서 두 명이 동시에 조작해야 해서 버튼 배치와 화면 구성이 복잡해졌습니다.

해결 방법

플레이어별 조작 영역을 분리하고, 게임 보드는 중앙에 배치해서 한 기기에서도 두 명이 플레이할 수 있도록 구조를 정리했습니다.

What I Learned

배운 점

이 프로젝트를 만들면서 JavaScript로 게임 상태를 관리하고, 점수와 승패를 계산하는 방법을 배웠습니다. 또 단순히 화면을 꾸미는 것이 아니라 사용자가 실제로 조작하기 편한 구조를 고민해야 한다는 것도 알게 되었습니다.

특히 모바일 웹 게임에서는 디자인뿐만 아니라 터치 입력, 화면 크기, 버튼 위치, 브라우저 기본 동작까지 함께 고려해야 한다는 점을 배웠습니다.
